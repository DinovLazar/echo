# Part 1 · Phase 04 · Code — Completion Report
**Date:** 2026-06-24 · **Outcome (one line):** The board can now record a run of moves, **fold** it into a grey echo, and replay that echo in exact lockstep with the shared turn counter — so for the first time there are several of you on the board at once.

## 1. What shipped (plain language)
ECHO now has its keystone mechanic. As you move, the app records your run; pressing **Fold** banks that run as a translucent grey "echo" and rewinds the room to the start (you back on the centre cell, the turn counter back to 0, a fresh empty run). From then on the echo retraces your exact path one step per turn, locked to the same clock you move on, and stands still on its last tile once it runs out of moves — so the board fills with one solid black "you" and several grey past-selves. A small throwaway debug strip below the board has **Fold**, **Clear** (wipe everything back to a pristine room), and a live readout of the turn number and echo count. There is deliberately **no collision yet** (you can stand on or walk through an echo) — that lands next phase.

## 3. Decisions I made during this phase
- **Echo replay model — echo stores `[Direction]`, position is a pure function of (start, recording, turn).** *Why:* makes replay trivially exact/testable and means all echoes step automatically as `turn` ticks (no imperative per-echo stepping to desync); the "exhausted echo stands still" rule falls out of the `min(t, k)` clamp for free. *Rejected:* storing per-turn absolute cells (redundant derived data) or mutating a per-echo cursor each move (desync risk). **Needs Decisions entry: YES → D-014.**
- **Fold rewinds the whole board to turn 0; folding an empty run is a no-op.** *Why:* matches Plan §14's "time rewinds to turn zero," makes one fold = one more self + clean restart, and the empty guard keeps the control idempotent/safe. *Rejected:* fold-without-rewind; allowing zero-move echoes. **Needs Decisions entry: YES → D-015.**
- **Collision intentionally omitted (pass-through echoes) in 1.04.** *Why:* the brief scopes collision to 1.05; keeping it out keeps this phase provably correct in isolation. *Rejected:* adding collision early (couples two phases) or dead half-measures. **Needs Decisions entry: YES → D-016.**
- **`GameState` ownership lifted from `BoardView` to `ContentView`; debug bar lives in `ContentView` taking an injected `let state`.** *Why:* the Fold/Clear controls must act on the same model the board renders, and the brief says keep them out of `BoardView`; a plain `let` injection is the smallest correct Observation wiring (anticipated by the 1.03 report). *Rejected:* keeping state in `BoardView`; an environment object (more ceremony than a one-screen app needs); an overlay bar (would intercept input). **Needs Decisions entry: YES → D-017.**
- **`fold()` returns `@discardableResult Bool`** (did-fold), mirroring `move(_:)`. *Why:* cheap signal a future undo/feedback path can use; harmless now. *Rejected:* `Void`. Decisions entry: NO (small, additive; noted under D-015).
- **Echo placeholder visuals** — same size as the player (76% of cell), `Color(white: 0.5).opacity(0.35)` fill, thin `Color(white: 0.25).opacity(0.55)` outline, `.easeInOut` slide. *Why:* satisfies "translucent grey rounded square, thin outline, ~35% fill"; explicitly a placeholder for the 2.01 design pass. Decisions entry: NO.
- **`Echo` imports `Foundation`** (for `UUID`) and is `Identifiable` with a `UUID id`. *Why:* stable identity lets `ForEach` animate each grey square independently. Decisions entry: NO.

## 7. Risks, follow-ups, what the next phase needs to know
- **Owed on Lazar's full Xcode (nothing below was done here):** open in Xcode 27 and **⌘R** the iOS Simulator to confirm the board, echoes, and debug bar render and fold/clear behave; **⌘U** to run the real XCTest suite; capture a real screenshot; and **pin the exact Xcode 27 / iOS SDK build numbers** in `00_stack-and-config.md` (paste `xcodebuild -version`).
- **Phase 1.05 (collision) builds directly on this:** the pure per-turn position functions (`Echo.position(start:turn:)` / `GameState.position(of:)`) are exactly what collision compares — "player's cell == any echo's cell at the same turn → dissolve and restart the current run; existing echoes persist." Recording already stops/records correctly; collision will need to clear `currentRun` and return the player to `start` *without* touching `echoes` (distinct from `fold` and `clearEchoes`).
- **Remove the debug bar in Parts 2–3** (D-017) and wire the *real* reset-run / step-back in Phase 1.07 — `Clear` is only a stand-in. Tracked in `current-state.md` Known issues.
- **Don't "fix" pass-through echoes** before 1.05 — it's intended (D-016).
- **Animation note:** fold rewinds `turn` to 0, so on a fold every echo (and the player) animates back toward `start` with the placeholder `.easeInOut`. That's acceptable per the brief ("a plain animation on fold/rewind is fine; do not tune it"); the weighty fold motion is Part 2 (2.02).
