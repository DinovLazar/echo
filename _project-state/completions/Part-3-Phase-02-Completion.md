# Part 3 · Phase 02 · Code — Echo Run (arcade survival mode) · Completion Report
**Date:** 2026-06-28 · **Outcome (one line):** ECHO now has a second mode — **Echo Run**, an endless arcade survival run against an ever-thickening crowd of your own delayed-shadow echoes, scored by turns survived — built as a separate, additive engine on the existing move/replay core.

## 1. What shipped (plain language)
ECHO is now **two games from one verb**. Alongside the finished 20-room campaign there is **Echo Run**: you move on an open 9×9 board, and every 8 turns a grey "shadow" of your past self spawns at the centre and retraces the exact path you've walked, trailing behind you. Touch any shadow and the run ends; you survive as long as you can for a high score. Swiping into the wall is a "stall" — you hold your ground but the world still advances, so you can't just camp forever (the trailing shadows close in). It looks, sounds, and feels like the campaign — same board, same motion, same death dissolve, same ticks and taps — because it reuses all of that. It is entirely additive: the campaign engine and all 20 rooms are untouched.

## 3. Decisions I made during this phase
- **`BoardView` reuse vs a dedicated arcade view → built a dedicated `EchoRunView`.** `BoardView` is tightly coupled to `GameState`/`Level` (it reads walls/doors/switches/hazards, takes a `GuidanceController`, and fires the fold choreography on echo-count change) — none of which Echo Run has. Bending it would have meant making `EchoRunState` masquerade as `GameState`. The brief explicitly sanctions this fallback ("if it is too coupled … build a small dedicated arcade board view that reuses `Theme`/`Motion`/`BoardEffects`"). · Alternative rejected: adapt/parameterise `BoardView` — more coupling, more risk to the campaign view. · Recorded in **D-058**'s note. Needs `ECHO-Decisions.md` entry: **already covered (D-058).**
- **Injectable `spawnPeriod` (default 8) as a test seam.** `EchoRunState.init(size:spawnPeriod:)` defaults `spawnPeriod` to 8 (the D-057 cadence). Tests inject a small period (1) to construct land-on/swap collisions in a few turns without driving the full 8-turn cadence; production always uses 8, exercised by `testSpawnCadenceFiresEveryEighthTurn`. · Alternative rejected: hand-derive collisions through the real 8-turn cadence on a 9×9 — tedious and far less readable. · Recorded in **D-058**'s note. Needs its own entry: **NO** (a test affordance whose default is the decided value).
- **Score includes the fatal turn (`score == turn`).** The brief says "the turn count reached when the run ends." The fatal turn ticks the counter before the collision is detected, so a death on the move that makes turn 16 scores 16. · Alternative: score the last *survived* turn (15) — rejected as a re-interpretation of the brief's wording. · Pinned by `testCrossPathsDeathWhenAShadowSwapsTilesWithThePlayer` (death at turn 3 → score 3). Needs an entry: **NO** (it implements the brief's stated definition).
- **Stalls append nothing to the recording.** The brief says a stall "the trail stops growing," so a stall advances the turn but does not record a move; the player stays the tip of the recorded path. · Alternative: record a "wait" — `Direction` has no such case and it would desync the echo trail from the spatial path. Needs an entry: **NO** (it implements D-057).
- **Deferred-death (predict → dissolve → commit), mirroring the campaign.** `EchoRunView` predicts the fatal turn via `EchoRunState.pendingDeath(for:)` and only commits in `finishDeath()` after the dissolve, so the board freezes during the fizz exactly like the campaign. `testPendingDeathAgreesWithCommit` proves the predictor and the commit agree. Needs an entry: **NO** (an implementation detail of D-058's "reuse `BoardEffects`").
- **Doc slip corrected:** `file-map.md` was missing the `Part-3-Phase-01-Completion.md` line (the 3.01 report exists on disk). Added it (with a note) alongside the 3.02 line, per the file-map's "if the map and the repo disagree, fix it and note the slip" rule. Needs an entry: **NO** (a doc correction).
