## Summary (plain language)
- **Works now:** ECHO has a real, state-driven board. On the warm off-white ("paper") background sits a centered **7×7 grid of grey square cells** with a **solid black rounded-square player** on the **center** cell. You move the player one tile at a time: **swipe** in a cardinal direction steps that way, and **tapping a cell directly next to the player** (up/down/left/right — not diagonal) steps into it. The player **can't leave the grid** — an off-grid swipe or a tap on a far/diagonal cell does nothing. Each real move advances a hidden **turn counter** by one (no-ops don't). The square **slides** between cells (a plain placeholder animation, not the tuned motion yet). The throwaway hello-grid is gone.
- **Stubbed / not wired yet:** Everything past "move one square." No fold/record/replay, no echoes, no collision, no win/exit, no walls/switches/doors/hazards, no level JSON, no reset-run or step-back controls, no audio (Part 2), no haptics (Part 2), no accent color or invert mode. The turn counter is tracked but not shown on screen. The visual look is still deliberately pre-design "grey boxes on paper" (real palette + tuned motion are Part 2). `ECHO/Audio/` and `ECHO/Haptics/` still hold no code and are **not git-tracked while empty** (no `.gitkeep` — see D-012); `ECHO/Models/` is now populated and tracked. `Levels/` is still an empty placeholder.
- **Current phase:** 1.03 — **done in this environment to the limit the toolchain allows.** The move model and board view are written; the move logic is **verified correct by a standalone harness** that runs every test scenario (all 18 checks pass) against the real model sources, and all sources **type-check clean** under both the app target's and the test target's actor-isolation regimes. The iOS Simulator build, the on-device ⌘R run, and the XCTest run (⌘U) still need **Lazar's full Xcode** — this Code environment has only the Command Line Tools (no `xcodebuild`/`simctl`, and the SwiftUI/`#Preview` macro plugins are absent), so those are owed on his Mac.
- **Next:** Phase 1.04 — fold / record / replay (the keystone). It records sequences of exactly these moves and replays them as grey echoes locked to the shared turn counter built here.

## Detail
- **`ECHO/App/ECHOApp.swift`** — `@main` SwiftUI `App`; opens a `WindowGroup` containing `ContentView`. Unchanged this phase. Real.
- **`ECHO/App/ContentView.swift`** — root view: a `ZStack` of the warm off-white paper (`Color(red: 0.96, green: 0.94, blue: 0.89)`, inline) with `.ignoresSafeArea()` (full-bleed) behind `BoardView()` (inset to the safe area). Now composes the real board instead of the hello-grid. Real.
- **`ECHO/Views/BoardView.swift`** — the real board (replaces `HelloGridView`). A `GeometryReader` sizes square cells to 82% of the smaller available dimension (`cell = available / max(width, height)`, so the default 7×7 matches the old hello-grid sizing and stays square for any future non-square board) and centers the board. Draws the grey lattice (`Rectangle`s with thin `Color(white: 0.7)` 1 pt borders) with **per-cell tap** targets, and the player as a black `RoundedRectangle` filling 76% of its cell, positioned at its cell center via `.position` and animated with a plain `.easeInOut` on `state.player` (placeholder slide). Input: a board-level `DragGesture(minimumDistance: 20)` whose dominant axis picks the swipe direction, and per-cell `onTapGesture` that routes through `Direction(from:to:)`. Both call `GameState.move(_:)`. Real.
- **`ECHO/Models/GameState.swift`** — `@MainActor @Observable final class` (see D-013). Holds `width`/`height` (params, default 7×7), `player` (`private(set)`, default = center cell), and `turn` (`private(set)`, starts 0). `init(width:height:start:)` makes dimensions **and** the start cell parameters (start defaults to center) for Phase 1.06. `move(_:) -> Bool` (`@discardableResult`): clamps to the board (off-grid = no-op), and on a legal move updates `player` and increments `turn` by exactly one. Pure, view-independent, unit-tested. Real.
- **`ECHO/Models/GridCoordinate.swift`** — `nonisolated struct GridCoordinate: Equatable, Hashable, Sendable` (`row`, `column`; origin top-left). The coordinate the whole game speaks in. Real.
- **`ECHO/Models/Direction.swift`** — `nonisolated enum Direction: CaseIterable` (`up/down/left/right`) with `offset` (row/col delta) and a failable `init?(from:to:)` that returns the direction between two orthogonally-adjacent cells (`nil` for same/diagonal/farther — the tap rule). Real.
- **`ECHO/Resources/Assets.xcassets`** — asset catalog with an empty `AppIcon` set (no artwork) and an empty `AccentColor` set (system default). Unchanged, intentionally bare.
- **`ECHO/Audio/`, `ECHO/Haptics/`** — reserved for Part 2; no files, **not git-tracked while empty** (no `.gitkeep`, see D-012); reappear when populated.
- **`ECHOTests/ECHOTests.swift`** — `@MainActor final class ECHOTests: XCTestCase` (see D-013) with the real move-model coverage: default board (7×7, center, turn 0); a legal step in each of the four directions; an off-grid no-op at each of the four edges; the turn counter incrementing by one per committed move and unchanged on no-ops; and the `Direction(from:to:)` tap rule (adjacent → direction; same/diagonal/far → nil). Replaces the trivial `2 + 2` test. **Not yet run via ⌘U here** (no XCTest in CLT); logic verified by harness — see Build status.
- **`Levels/`** — empty (`.gitkeep`); room JSON starts in Phase 1.06, not yet wired into the build.
- **`docs/design-handovers/`** — reserved, empty (`.gitkeep`). No design handover for this phase by design (pre-design "grey boxes").
