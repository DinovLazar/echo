## Summary (plain language)
- **Phase 2.05 (most recent change):** ECHO now has **haptics** — a touch tap paired with each of the four game events that already exist: a light **selection** tick on a committed step, a medium **impact** on a fold, a soft **error** notification when an echo/hazard catches you, and a **success** notification when you reach the exit. New `ECHO/Haptics/HapticsManager.swift` (a `@MainActor final class`) owns the three system `UIFeedbackGenerator`s and exposes exactly four methods — `step()` / `fold()` / `collision()` / `win()` — mapped to `selectionChanged()` / `impactOccurred()` / `notificationOccurred(.error)` / `notificationOccurred(.success)`. A `var isEnabled` (default `true`) gates **all** output, pre-warming included — the single binding point the Phase 2.06 Settings haptics-toggle will use (no toggle UI or persistence built here). It is **presentation-only**: the engine, models, level JSON, and the deterministic rules are untouched. `ContentView` owns the manager (`@State`, exactly like `AudioManager`) and pre-warms it at launch; `BoardView` fires the four taps from the same `commitMove` / `triggerFold` paths that already fire the audio and the motion, so each tap lands with its visible state change and (for the step and fold) its sound. One **additive, read-only** model signal was added — `MoveOutcome { stepped, died, won }` + `private(set) var lastMoveOutcome` on `GameState`, set only inside `move()` — with six focused tests; no existing method, behaviour, or test changed. Decision **D-049** records the approach (and corrects the brief's stale framing — see below).
- **Where the brief was stale:** the 2.05 brief was written against a **Phase 1.06** snapshot (it assumed motion/particles/audio did not exist yet, that haptics paired with "placeholder" visuals, that the triggers wired from `ContentView` reading `lastMoveOutcome`, and that the next decision id was "D-028"). The repo is actually at **2.04**: those visuals/audio shipped, the decision log was at D-048 (so this is **D-049**, not D-028 — long taken by step-back), and all input + effects fire from `BoardView`, where a fatal step never calls `state.move()` (it predicts the death and defers the mutation to `finishDeath()`). The implementation adapts to the live code (live code wins, per CLAUDE.md): wired in `BoardView` alongside the audio, decoupled from `lastMoveOutcome` for the deferred-death path. The taps now land on the **real** 2.02/2.03 visuals and 2.04 audio (better than the brief's placeholder expectation). Full rationale in **D-049** and the Phase 2.05 completion report §3.
- **Phase 2.04 (prior change):** ECHO **makes sound** — every successful step plays a soft pitched-percussion tick, one per moving entity, fired at one shared host time so a busy turn is a **chord** and a recorded run a **rhythm**; plus restrained **fold** / **death** / **solve** event sounds timed to the 2.03 beats. `ECHO/Audio/AudioManager.swift` owns one `AVAudioEngine` + a pool of 16 player nodes and a sound bank **synthesized once at startup** (no bundled audio). `var isEnabled` gates playback (the 2.06 sound-toggle hook).
- **Phase 2.03 (prior change):** the two signature moments became legible *events* — **folding** plays the §6c choreography (hit-pause → grid ripple → the grey echo peeling off and rewinding), **dying** plays the §6d dissolve (glide → freeze → particle **fizz** → red vignette → instant restart). `ECHO/Views/BoardEffects.swift` holds the Canvas overlay.
- **Phase 2.02 (prior change):** the board became the real ECHO — every element drawn to the locked Phase 2.01 design in two palettes (Light + Invert) from `ECHO/Theme/Theme.swift`; pieces **glide** between cells via the named curves in `ECHO/Theme/Motion.swift`.
- **Engine + tests untouched as rules; one additive model signal:** `move()`/`fold()`/`restartRun()`/`stepBack()`/`clearEchoes()`/`playerCollides()` keep their exact signatures, return values, and behaviour. The only model change is the additive `lastMoveOutcome` signal (set inside `move()`, read by nothing in the engine), plus its six new tests. No turn/collision/win/replay rule, JSON format, or existing test changed.

Custom audio-synced `CHHapticPattern`s are deferred (D-049). Real.
