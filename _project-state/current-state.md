## Known issues
- **Resolved (D-012):** the first Xcode build reported `Multiple commands produce …/ECHO.app/.gitkeep` (1 error + 2 "duplicate output file" warnings) because Xcode's file-system synchronized groups copy `.gitkeep` into the app bundle as a resource, and the three identically-named `.gitkeep` files under `ECHO/` (Models, Audio, Haptics) collided. Fixed by removing those three files; after a Clean Build Folder the project built and ran on the Simulator. **Repo rule:** never place a `.gitkeep` (or any duplicate-named throwaway) inside `ECHO/` or `ECHOTests/`. (The Phase 1.01 report's assumption that synchronized groups ignore dotfiles was wrong.)
- **Toolchain not yet pinned:** `00_stack-and-config.md` still owes the exact Xcode 27 / iOS SDK build numbers (verified runtime so far: iOS 27.0 Simulator, iPhone 17 Pro). The trivial test (⌘U) has not been re-run.
