## Summary (plain language)
- **Works now:** Everything 1.03/1.04 had (a real 7×7 board, a solid black player you move one tile at a time by swipe or adjacent-cell tap, off-grid moves are no-ops, a hidden turn counter ticks one per committed move, and the keystone **fold** mechanic: pressing **Fold** banks your run as a permanent grey **echo** and rewinds the room to the start, after which every echo replays its recorded moves in lockstep with the shared turn counter and an exhausted echo stands still on its last tile) **plus the rule that finally makes echoes dangerous.** From this phase on, **touching one of your past selves dissolves present-you and restarts the run you are walking**: after any committed step the player is checked against every echo, and if you've touched one — you **land on** its tile this turn, or you and it **trade the same pair of adjacent tiles** — you snap back to `start`, the turn resets to 0, and your current (in-progress) run is discarded. Crucially, **all your folded echoes survive a death and keep replaying** — only the live run you were walking is thrown away. A throwaway **debug bar** below the board still has **Fold**, **Clear** (wipe to pristine), and a live `turn N · echoes M` readout.
- **Collision is a model rule, not a view trick:** the test lives in `GameState` (pure, view-independent, directly unit-testable). Because the model is `@Observable`, the board reacts to a death for free — `turn` resets to 0 and `player` to `start`, so the player and every echo slide back to their starts with the existing `.easeInOut`. This phase's death is an **immediate snap-back** (no death "fizz", no freeze-frame — those are Phase 2.03).
- **Cross-paths clause is written but dormant against echoes:** the collision predicate implements both land-on **and** cross-paths (swap), per **D-018**, because it is the correct general "did present-you touch a thing that moved this step" test. The swap clause **cannot currently fire against an echo** — the player and every echo share an origin and a one-step-per-turn cadence, so at any given turn they are always the same checkerboard parity, while a swap needs opposite-parity adjacent tiles. It exists for the independently-moving hazards of **Phase 1.06**, which can cross paths; it is covered by a predicate-level test now.
- **Stubbed / not wired yet:** the real **reset run** and **step back** controls (Phase 1.07 — the debug *Clear* is only a stand-in; the death restart `restartRun()` is the exact operation 1.07's reset-run will call), walls/switches/doors/hazards, the exit and win detection, level JSON (Phase 1.06), audio (Part 2), haptics (Part 2), the real palette/accent/invert mode, and tuned "weighty" motion (Part 2). The slide is still a plain `.easeInOut` placeholder. `ECHO/Audio/` and `ECHO/Haptics/` still hold no code and are **not git-tracked while empty** (no `.gitkeep` — see D-012). `Levels/` is still an empty placeholder.
- **Current phase:** 1.05 — **done in this environment to the limit the toolchain allows.** The collision predicate (`GameState.playerCollides(previousPlayerCell:newPlayerCell:turn:)`), the death restart (`GameState.restartRun()`), the `move(_:)` wiring, the stale-comment updates in `BoardView`, and the extended XCTest suite are written; the deterministic logic is **verified correct by a standalone harness** that runs every test scenario (all 1.03/1.04 checks plus the new 1.05 collision/restart checks) against the real model sources (**134/134** checks pass), and all sources **type-check clean** under both the app target's and the test target's actor-isolation regimes (the real test source also type-checks against the model API via an XCTest shim). ## Known issues
- **Toolchain not yet pinned:** `00_stack-and-config.md` still owes the exact Xcode 27 / iOS SDK build numbers (verified runtime so far, from Lazar's 1.02 build: iOS 27.0 Simulator, iPhone 17 Pro).
- **⌘U not run in CI/this env:** XCTest isn't available under Command Line Tools, so the suite is verified here only by an equivalent standalone harness (134/134 pass) plus a shim type-check of the test source. Lazar should run ⌘U once in Xcode to confirm against the real XCTest host.
- **Death is an immediate snap-back (intended for this phase):** because `move(_:)` performs the death restart synchronously (the mandated, pure, testable design), SwiftUI only observes the post-restart state, so the player slides from its pre-move cell back to `start` rather than visibly pausing on the tile it touched. When the collision is on the first step from `start`, that reads as "you didn't move." The **visibly-sequenced touch + death "fizz" + freeze-frame are owed to Phase 2.03** — don't add a timer/delay to the turn engine to fake it earlier (it would compromise determinism).
- **Cross-paths clause dormant against echoes:** the collision predicate's swap branch cannot fire against an echo today (parity-locked — see Summary / D-018); it is exercised only by a direct predicate-level test and first becomes reachable with Phase 1.06's independently-moving hazards. It is intended, not dead — don't remove it.
- **Throwaway debug bar is live:** the `Fold`/`Clear`/readout strip in `ContentView` is deliberately unstyled temporary UI (D-017) and **must be removed in Parts 2–3**; `Clear` is a debug stand-in, not the real reset run (Phase 1.07, which should call `restartRun()`).
- **Repo rule (D-012):** never put a `.gitkeep` (or any duplicate-named throwaway) inside `ECHO/` or `ECHOTests/` — synchronized groups copy it into the bundle and identical names break the build.
- `AppIcon` has no artwork yet (placeholder); a real icon is a later design task.
- Visuals are intentionally pre-design ("grey boxes on paper"); the real palette, accent, invert mode, and tuned motion are Part 2 (2.01/2.02).
