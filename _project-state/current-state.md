## Summary (plain language)
- **Works now:** The empty board is now an actual puzzle. Everything 1.03–1.05 had (a state-driven board you move a black player around one tile at a time by swipe or adjacent-cell tap, off-grid moves are no-ops, a hidden shared turn counter, the **fold** mechanic that banks a run as a grey **echo** and rewinds to turn 0 with every echo replaying in lockstep, and the rule that **touching a past self dissolves present-you and restarts the current run** while folded echoes survive) **plus the contents that make a room a room**, all loaded from small per-room **JSON files** in `Levels/`:
  - **Walls** block the player (a move into a wall is a no-op, exactly like off-grid — no turn tick, no record).
  - **Switches + doors:** a switch is *held* at a turn iff the player or any echo is on its cell; a door is *open* iff all its `heldBy` switches are held (AND) — both **derived per turn from positions**, not stored. A closed door blocks the player like a wall; **doors never kill**. The canonical solve works: fold an echo onto a switch so it holds a door open while present-you walks through.
  - **Hazards:** independently-moving lethal cells that patrol an authored path (one step/turn), **loop by default**, and **kill present-you on contact** — land-on **or** cross-paths/swap. They are not solid (you can step onto one — and die) and don't hold switches.
  - **Win:** reaching the exit **alive** sets a win flag and locks input; **collision is evaluated before win** on the same step, so a tile that is both exit and lethal is a death.
  - **Echo budget:** each room carries an `echoBudget`; `fold()` is refused at the cap.
  - A throwaway **debug bar** now has **Fold / Clear / Next** plus a `turn N · echoes M/budget B` readout and a **"Solved ✓"** stand-in once the exit is reached. **Next** cycles the three proof rooms.
- **The swap clause is now live (D-022):** the collision predicate's cross-paths branch (written in 1.05, D-018, but parity-locked against echoes) **fires for the first time** against a hazard that trades tiles with the player — covered by a test.
- **Everything stays a model rule, not a view trick:** walls/switches/doors/hazards/win all live in the pure, `@MainActor @Observable` `GameState` and are directly unit-tested. The board reacts to a death/win for free via Observation. Death is still an **immediate snap-back** (no death "fizz"/freeze — Phase 2.03).
- **Three proof rooms** (`Levels/p1-06-a/b/c.json`) each exercise one element (walls / switch+door / hazard) and are **verified solvable** (the standalone harness loads each from disk and plays a winning line; for the hazard room it also confirms the naive fast crossing dies and a 2-step retiming wins).
