## Summary (plain language)
- **Phase 3.02 (most recent change) — a whole new mode, zero risk to the campaign.** ECHO now has **two modes from one verb**: the 20-room campaign, and **Echo Run**, an arcade survival mode. In Echo Run you move on an open **9×9** board (start at the centre); **every 8 turns a new "delayed shadow" echo spawns at the centre and retraces the exact path you have walked**, trailing you by its spawn-turn offset. Touching any shadow ends the run. A swipe into the board edge is a **stall** — you hold, but the turn still advances and every shadow steps, so stalling is self-limiting (the trailing shadows close on a stalled head). **Score = turns survived**; the best is saved. It reuses the campaign's look, motion, death dissolve, audio ticks, and haptics, so it looks and feels like ECHO.
  - **It is purely additive (D-058).** The campaign engine and every shared value type — `GameState`, `Echo`, `Hazard`, `Level`, `LevelLoader`, `GridCoordinate`, `Direction` — are **byte-for-byte unchanged**, as is every `Levels/*.json` and every existing test. Echo Run only **adds**: a new engine, a new view, a new test file, two `SettingsStore` additions, and an interim entry in `ContentView`. `SWIFT_DEFAULT_ACTOR_ISOLATION = MainActor` untouched (D-013).
  - **The new engine `EchoRunState`** (`ECHO/Models/EchoRunState.swift`, `@MainActor @Observable`) implements the mechanic (D-057): the 9×9 board, one continuous never-folded `recording: [Direction]`, the shared `turn`, the live shadows, `move(_:)` with the edge-swipe stall, the every-8-turns spawn, **land-on OR cross-paths** death (the swap branch is **live** here — a stall breaks the shared-parity lock that makes it dormant in the campaign), `score` (= turns survived), and `reset()`. Shadow positions reuse the campaign's pure `Echo.position(start:turn:)` (D-014). New pure value types `RunEcho` / `RunOutcome` / `PendingDeath` are `nonisolated` + `Sendable`.
  - **The arcade screen `EchoRunView`** is a small **dedicated** board (not `BoardView`, which is coupled to `GameState`/`Level`) that reuses the shared feel layer — `Theme`/`BoardMetrics`, `Motion` (lockstep slide + player squash), `BoardEffects` (the death freeze + particle fizz), `AudioManager` (the per-move tick chord: the player plus each stepping shadow), `HapticsManager` (`step()`/`collision()`) — and the campaign's player/echo styling. It shows a **live score**, **defers** a fatal turn so the death dissolve plays over a frozen board (predict via `pendingDeath` → play → `finishDeath()` commits + saves the high score + reveals game-over), and presents an interim **game-over → instant Retry** panel.
  - **High score persists** through the existing `SettingsStore` (D-058): a new `Int` `echoRunHighScore` (key `echoRun.highScore`, default 0 — the Plan §6 second `UserDefaults` value) + `recordEchoRunScore(_:)` (keep-the-best). No second persistence system.
  - **Tests:** `ECHOTests/EchoRunTests.swift` (new, **+15 methods**) drives the real arcade engine: centre start on 9×9; spawn cadence fires at turns **8/16/24**; each shadow retraces the exact path at given turns; the stall (player holds, turn advances, shadows step, recording unchanged); **land-on** death (a trailing shadow catches a stalled head at turn 12) and **cross-paths/swap** death (turn 3); score = turns survived incl. stalls; input locked after death; a clean `reset()`; `pendingDeath` agrees with the commit; and the high-score round-trip + keep-best. **Verified run** (the ⌘U-equivalent harness): **109 test methods / 1596 assertions, 0 failures** (was 94 / 1419 at the 3.01 tip — exactly **+15 methods / +177 assertions**, all the new arcade coverage).
