## Detail
- **`ECHO/App/AppRoute.swift`** — **new (Phase 3.03).** `nonisolated enum AppRoute: Equatable, Sendable`: `title` / `mainMenu` / `levelSelect` / `room(String)` / `echoRun` / `settings`. The single source of truth for the current screen (D-059); `room` carries the `Campaign.roomIDs` id.
- **`ECHO/App/ContentView.swift`** — **Phase 3.03 (rewritten as the root router).** Holds one `@State route: AppRoute` and switches `body` on it. Owns the session-long services (`SettingsStore`, `GuidanceController`, `AudioManager`, `HapticsManager`), derives the palette from `settings.invertEnabled`, and lifts `.environment(\.theme,…)`/`.tint` to the root. Builds the campaign `GameState` for the open room fresh in `openRoom(_:)` (`LevelLoader`, bare-board fallback) and holds it (`roomState`) so it survives re-renders; gives `RoomView` `.id(roomID)` so the win overlay can't leak across rooms. Light ~200 ms route fade (`.animation(Motion.guidanceIn, value: route)`); at launch sets `audio`/`haptics` `isEnabled` from the prefs, `start()`/`prepare()`s both, keeps them synced via `.onChange`. All interim scaffolding deleted (D-059).
- **`ECHO/Models/Campaign.swift`** — **new (Phase 3.03).** `nonisolated enum Campaign` — the room-order/identity single source of truth (`roomIDs` + `contains`/`index`/`number`/`next`/`firstUnsolved`). Free pick, no gating (D-060). Rides `Models/*.swift`, so it is in the headless test module — fully unit-tested.
- **`ECHO/Views/TitleView.swift`** — **new (Phase 3.03).** Launch screen: paper field, large ink "Echo," the player/echo motif, "tap to begin"; whole-screen tap → Main Menu. `Theme` tokens only; inherits invert.
- **`ECHO/Views/MainMenuView.swift`** — **new (Phase 3.03).** A small "Echo" wordmark + three `ControlButtonStyle` buttons (Campaign / Echo Run / Settings). Nothing else.
- **`ECHO/Views/LevelSelectView.swift`** — **new (Phase 3.03).** The 4×5 grid over `Campaign.roomIDs`: room number + the exit-ring motif (hollow ink = unsolved / filled ink disc = solved), the muted gold accent on **only** the one next-to-play tile, a "Menu" back chevron top-left, every tile tappable → opens that room. - **`ECHO/Views/RoomView.swift`** — **new (Phase 3.03).** The campaign room screen: reuses `BoardView` as-is and wraps it in the 2.07 HUD layout (back chevron · centred room number · turn/echoes readout) + a control row (Fold / Step back / Reset run) **gated on the mirrored input lock** (D-059). Takes the `GameState` passed in; fresh identity per room. Calls `guidance.enterRoom` on appear; on `state.hasWon` marks the room solved and shows the **win overlay** (ink closed-ring solved mark; "Next room" in `solvedGreen` → `Campaign.next` / "Level select"; final room → "Campaign complete" + only "Level select").
