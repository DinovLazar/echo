## Summary (plain language)
- **Works now:** Everything 1.03 had (a real 7×7 board, a solid black player you move one tile at a time by swipe or adjacent-cell tap, off-grid moves are no-ops, a hidden turn counter ticks one per committed move) **plus the keystone fold mechanic.** As you move, the app now **records** your run (the ordered list of moves since the last fold). Pressing **Fold** banks that run as a permanent **grey echo** and rewinds the whole room to the start — player back on the centre cell, turn back to 0, a fresh empty run — and every existing echo returns to its own start. From then on each echo **replays its recorded moves in perfect lockstep with the shared turn counter**: at turn `t` an echo sits where you were after your first `t` moves, and once it runs out of moves it **stands still on its last tile**. So for the first time there can be several of you on the board — one solid black "me," several translucent grey "thems." A throwaway **debug bar** below the board has **Fold**, **Clear** (wipe to pristine), and a live `turn N · echoes M` readout.
- **No collision yet (intended):** touching an echo does **nothing** in 1.04 — the player and echoes may share a tile and you walk straight through them. That is correct for this phase; collision (same tile, same turn → you dissolve and the current run restarts) is **Phase 1.05** (D-016).
- **Stubbed / not wired yet:** the real **reset run** and **step back** controls (Phase 1.07 — the debug *Clear* is only a stand-in), walls/switches/doors/hazards, the exit and win detection, level JSON (Phases 1.05–1.06), audio (Part 2), haptics (Part 2), the real palette/accent/invert mode, and tuned "weighty" fold motion (Part 2). The fold/rewind currently uses a plain `.easeInOut` placeholder slide, same as the player. `ECHO/Audio/` and `ECHO/Haptics/` still hold no code and are **not git-tracked while empty** (no `.gitkeep` — see D-012). `Levels/` is still an empty placeholder.
- **Current phase:** 1.04 — **done in this environment to the limit the toolchain allows.** The `Echo` type, the extended `GameState` (record/fold/clear + per-echo position), the echo rendering, and the debug bar are written; the deterministic fold/replay logic is **verified correct by a standalone harness** that runs every test scenario (all **39/39** checks pass) against the real model sources, and all sources **type-check clean** under both the app target's and the test target's actor-isolation regimes (the real test source also type-checks against the model API via an XCTest shim). The iOS Simulator build, the on-device ⌘R run, and the XCTest run (⌘U) still need **Lazar's full Xcode** — this Code environment has only the Command Line Tools (no `xcodebuild`/`simctl`, and the SwiftUI `@State`/`#Preview` and XCTest macro/framework plugins are absent), so those are owed on his Mac.

## Detail

- **`ECHO/App/ContentView.swift`** — root view; now **owns** the `GameState` (`@State private var state`) so the board and the debug bar act on one model (D-017). Lays out a `ZStack` of full-bleed warm off-white paper (`Color(red: 0.96, green: 0.94, blue: 0.89)`, inline) behind a `VStack` of `BoardView(state:)` above a **throwaway debug bar** (`Fold` → `state.fold()`, `Clear` → `state.clearEchoes()`, and a `turn N · echoes M` readout). The bar sits below the board, clear of its gesture area, and is commented as temporary UI to remove in Parts 2–3. Real.
- **`ECHO/Views/BoardView.swift`** — the board, now taking an injected `let state: GameState` (ownership moved to `ContentView`). Same `GeometryReader` sizing (square cells at 82% of the smaller dimension, centered), the grey lattice with per-cell tap targets, and the black `RoundedRectangle` player (76% of its cell) positioned via `.position` and animated with a plain `.easeInOut` on `state.player`. **New:** an `echoes(cell:)` layer drawn **between** the lattice and the player — a `ForEach(state.echoes)` of translucent grey rounded squares (mid-grey ~35% fill, thin darker outline, same size as the player), each positioned at `state.position(of: echo)` and animated with `.easeInOut` on that position, with hit-testing off so taps fall through (no collision — D-016). The player always draws last/on top. Board-level `DragGesture(minimumDistance: 20)` and per-cell `onTapGesture` are unchanged. Real.
- **`ECHO/Models/GameState.swift`** — `@MainActor @Observable final class` (D-013). Holds `width`/`height` (params, default 7×7), the stored **`start`** cell (default centre), `player` (`private(set)`), `turn` (`private(set)`, starts 0), and **new** `private(set)` `currentRun: [Direction]` and `echoes: [Echo]`. `move(_:) -> Bool` is unchanged except a committed move now **appends its direction to `currentRun`** (no-ops still change nothing). **New:** `position(of:) -> GridCoordinate` (per-echo current cell, a pure function of `start`+moves+`turn`); `fold() -> Bool` (banks `currentRun` as an `Echo`, then rewinds player→`start`, `turn`→0, empties the run; empty-run fold is a no-op — D-015); `clearEchoes()` (wipe echoes + run, player→`start`, `turn`→0 — debug only, Plan §14 point 6). Pure, view-independent, unit-tested. Real.
- **`ECHO/Models/Echo.swift`** — **new.** `nonisolated struct Echo: Identifiable, Equatable, Sendable` with a stable `id: UUID` (so SwiftUI animates each grey square independently) and the recorded `moves: [Direction]`. Its `position(start:turn:)` applies the first `min(turn, moves.count)` moves from `start`, so an exhausted echo stands still on its last tile (D-014). Matches the `nonisolated … Sendable` discipline of `GridCoordinate`/`Direction`. Real.
- **`ECHO/Models/GridCoordinate.swift`** — `nonisolated struct GridCoordinate: Equatable, Hashable, Sendable` (`row`, `column`; origin top-left). Unchanged this phase. Real.
- **`ECHO/Models/Direction.swift`** — `nonisolated enum Direction: CaseIterable` (`up/down/left/right`) with `offset` and failable `init?(from:to:)`. Unchanged this phase. (Implicitly `Equatable`/`Hashable`/`Sendable` — relied on by `Echo`/tests.) Real.
