## Summary (plain language)
- **Phase 2.03 (most recent change):** the two signature moments became legible *events* instead of instant pops, presentation-only (engine/models/levels/tests untouched). **Folding** now plays the handover §6c choreography — a 50 ms hit-pause, a quiet grid ripple across the hairlines, and the grey echo of the just-walked run **peeling off and rewinding** from the run-end back to the start square (ink → echo-grey, 32 → 28 pt). **Dying** (touching an echo or hazard) now plays the §6d dissolve — the player (and the echo it touched) **glide** onto the contact tile, a calm ~66 ms freeze, both selves **fizz** into soft SwiftUI-**Canvas** particles, a faint red vignette + one enemy-glow pulse — then the run restarts **instantly** with echoes intact. New `ECHO/Views/BoardEffects.swift` holds the Canvas overlay; `BoardView` detects a fold (echo-count rose) and **defers** a fatal step's model mutation so the dissolve can show the contact frame before `restartRun()`. Both effects theme in **both palettes**, zero new dependencies, **no SpriteKit**. Two approach decisions recorded: **D-043** (the fatal step glides before the freeze — no-teleport over the literal §6d) and **D-044** (the peel is realised as a rewind from the run-end, not an in-place fade). Reset / step-back / room-load still snap, as scoped.
- **Phase 2.02 (prior change):** the board stops being grey boxes and becomes the real ECHO. Every element is now drawn to the locked Phase 2.01 visual design — real colours (warm paper, ink, translucent echoes, **red diamond enemy**, **gold goal**), real geometry/strokes/opacities, soft glows and a player drop shadow — in **two full palettes (Light default + Invert)**. Pieces **glide** between cells instead of snapping: the player slides with a squash-and-stretch and a soft-snap landing, echoes slide in lockstep, and the hazard moves a touch heavier with an anticipation lean-in. This is presentation only — **the turn engine, the models, the JSON format, and the XCTest suite were not touched.**
- **Colour-token system (new):** `ECHO/Theme/Theme.swift` is the single source of every board colour. It defines both authoritative palettes for all handover §1a tokens (Light + Invert), defaults to Light, and exposes **one internal switch point** — the `\.theme` SwiftUI environment value — that the Settings phase (2.06) will bind to a real, persisted user toggle. No toggle UI and no persistence were built this phase (D-041). Geometry/stroke/opacity/shadow/glow metrics (§1b) live alongside in `BoardMetrics`, in points at the reference cell `C = 44`, scaled to the runtime cell so proportions hold on any device.
- **Named motion curves (new):** `ECHO/Theme/Motion.swift` implements the five handover §6a curves as reusable `Animation`s with the exact control points (`curve.standard`/`easeOut`/`easeIn`/`softSnap` as `timingCurve`s; `curve.decayShake` as `interpolatingSpring(stiffness: 320, damping: 14)`), plus the §1c motion tokens built from them. Only `step` (120 ms), `stepSnap` (40 ms) and `enemyStep` (140 ms) are wired into the board this phase; the fold/death/deny/guidance/trail tokens are defined for the later phases that own those beats.
- **`BoardView` rewritten to spec:** faint `tile.hairline` lattice (still the tap layer) under element layers drawn to handover §2 — walls (top-light gradient tile), exit (active-goal **gold ring + glow**, with the default ink-ring state also implemented), doors (one solid **closed** bar vs two faint **open** remnant stubs), switches (hollow **open** ring vs filled-gold **held** circle + glow), the enemy (**red diamond** = rotated rounded square, + inner core + outline + soft glow), translucent recede-behind **echoes**, and the fully-opaque **ink player** — the largest piece (32 pt vs the echo's 28 pt) and the **only shadow-caster**, so it always reads as "now" (§3). Grayscale identity holds: every element is still unique by shape + size + opacity with colour removed (D-041).
- **Step motion:** a committed move runs through a new view-side `commitMove(_:)` that reads the model's **public predicates** (no mutation) to choose how to *display* the result. A **survived step** is wrapped in `withAnimation(Motion.step)` so the player and every echo glide 120 ms `curve.standard` **in lockstep**, with the hazard retimed to 140 ms via a `.transaction` override; it also bumps a `stepTick` that fires the player's squash-and-stretch + soft-snap and each hazard's anticipation lean-in (SwiftUI `keyframeAnimator` scale tracks). A **fatal step, a fold, a reset, or a room load** is **not** wrapped, so it snaps on the next frame (restart = 0 ms, per §6d). The engine still decides everything; a mis-prediction could only mis-pick the animation, never the outcome.

It owns the active `ThemeMode` (the switch point) and injects the resolved palette into the environment. The throwaway debug bar gained an **Invert** button that flips the palette in place for on-device verification — a debug control like *Next*, explicitly **not** the real 2.06 Settings toggle. All existing debug controls (Fold / Step back / Reset run / Clear / Next / readout / Solved ✓) still work so the ten rooms can be navigated to view the new look.
