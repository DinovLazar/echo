## Summary (plain language)
- **Phase 2.04 (most recent change):** ECHO now **makes sound** — the signature feature. Every successful step plays a soft pitched-percussion tick, **one per moving entity** (you and each echo that stepped that turn), all fired at the **same instant** so a busy turn is heard as a **chord** and a recorded run as a **rhythm**; because echoes replay their moves in lockstep, a solved room re-voices itself into a little loop of percussion you composed by solving it. The four move directions map to four pitches from a fixed **pentatonic** set (up = higher, down = lower, left/right = the middle tones), so any combination of simultaneous ticks stays consonant. Three restrained event sounds round out the pass, each timed to the 2.03 animation beat already playing: a warm **fold** settle (on the §6c hit-pause onset), a calm muted **death** puff (on the §6d particle fizz), and a gentle ascending **solve** figure (as you reach the exit). It is **presentation-only** — the engine, models, level JSON, and the XCTest suite are untouched. New `ECHO/Audio/AudioManager.swift` owns one `AVAudioEngine` with a pool of player nodes and a sound bank that is **procedurally synthesized once at startup** (no bundled audio files — the public repo stays asset-free). `BoardView` fires the sounds from the same `commitMove`/`triggerFold`/`triggerDeath` paths the motion uses; `ContentView` creates the manager and starts it at launch. An `isEnabled` switch gates all playback — the single binding point the Phase 2.06 Settings sound-toggle will use (no toggle UI or persistence built this phase). Four decisions recorded: **D-045** (AVAudioEngine + player nodes + synthesized buffers), **D-046** (soft pitched percussion, pitch by direction), **D-047** (`.ambient` + `.mixWithOthers` session), **D-048** (full first audio pass, not step-ticks-only).
- **Phase 2.03 (prior change):** the two signature moments became legible *events* — **folding** plays the §6c choreography (50 ms hit-pause, a quiet grid ripple, the grey echo of the just-walked run **peeling off and rewinding** to the start square), and **dying** plays the §6d dissolve (the player and the echo it touched **glide** onto the contact tile, a ~66 ms freeze, both selves **fizz** into Canvas particles, a faint red vignette + one enemy-glow pulse, then an **instant** restart with echoes intact). `ECHO/Views/BoardEffects.swift` holds the Canvas overlay; `BoardView` defers a fatal step's model mutation so the dissolve can show the contact frame before `restartRun()`. Decisions D-043 (the fatal step glides before the freeze) and D-044 (the peel is a rewind from the run-end). Reset / step-back / room-load still snap.
- **Phase 2.02 (prior change):** the board became the real ECHO — every element drawn to the locked Phase 2.01 design in **two palettes (Light + Invert)** from the `ECHO/Theme/Theme.swift` colour-token layer; pieces **glide** between cells (player slide + squash-and-stretch + soft-snap; echoes in lockstep; the hazard heavier with an anticipation lean-in) via the named curves in `ECHO/Theme/Motion.swift`.
- **Audio is the only new subsystem:** `ECHO/Audio/AudioManager.swift` is a `@MainActor final class` that owns the `AVAudioEngine`, a pool of 16 `AVAudioPlayerNode`s into the main mixer, and the synthesized sound bank, and exposes `playStep(directions:)` / `playFold()` / `playDeath()` / `playSolve()` plus the `isEnabled` gate. A turn's ticks are scheduled at **one shared future host time** so they start sample-aligned (a chord, not a flam). The engine, nodes, and buffers are non-`Sendable` but owned and touched only on MainActor, so Swift 6 concurrency stays clean; per **D-045** there is deliberately **no** `AVAudioSourceNode` render block. Synthesis is fully deterministic (no randomness), matching the engine's character.
- **Engine + tests untouched (verified):** `git diff` shows **no change** to `ECHO/Models/*`, `Levels/*`, or `ECHOTests/*`. No turn/collision/win/replay rule, JSON format, or test changed — this phase only adds sound and the wiring to trigger it.

- **`ECHO/Audio/AudioManager.swift`** — **new this phase.** A `@MainActor final class` owning one `AVAudioEngine` + a pool of 16 `AVAudioPlayerNode`s connected to `mainMixerNode` in a fixed 44.1 kHz mono float format, plus the synthesized sound bank. `init()` builds the bank and the graph (cheap, side-effect-free — safe in `@State`/previews); `start()` configures the `AVAudioSession` (`.ambient` + `.mixWithOthers`, **iOS-only behind `#if os(iOS)`**), `prepare()`s and starts the engine, sets the master trim on the live mixer, and keeps the pool playing so each fire is a single `scheduleBuffer(at:)` at a shared future host time (sample-aligned chords). Triggers: `playStep(directions:)` (a chord, fired at commit), `playFold()` (hit-pause onset), `playDeath()` (offset to the §6d fizz via `Motion.Span.step + deathFreeze`), `playSolve()` (offset one step to arrival). `var isEnabled` (default `true`) gates everything — the only Settings hook (2.06).
